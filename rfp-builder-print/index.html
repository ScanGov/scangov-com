3. ScanGov Standards Compliance Requirements

The successful vendor must ensure the new website achieves full compliance with all applicable ScanGov Standards as outlined at https://standards.scangov.org. The following standards are mandatory:

3.1 Search Engine Optimization & Discoverability

  • Robots Policy: Implement valid robots.txt allowing appropriate access
  • Sitemap: Create and maintain XML sitemap with all website pages
  • Indexing: Ensure site is available for indexing by search engines
  • Canonical URLs: Implement preferred page URLs to avoid duplication
  • Descriptive Links: All links must have descriptive text
  • Meta Information: Proper page titles and descriptions

3.2 Accessibility Standards

  • WCAG Compliance: Meet WCAG 2.1 AA accessibility guidelines
  • Screen Reader Support: Implement accessibility properties for screen readers
  • Alternative Content: Provide alt text, captions, and transcripts for multimedia
  • Color Contrast: Ensure adequate contrast between text and background colors
  • Hidden Elements: Properly handle hidden elements for accessibility
  • Language Attributes: Use correct language attributes
  • Keyboard Navigation: Ensure full keyboard accessibility
  • Tables and Lists: Make all data tables and lists accessible
  • Descriptive Identifiers: Use clear, descriptive element identifiers

3.3 Performance Requirements

  • Page Load Speed: First Contentful Paint within 1.8 seconds
  • Interactivity: Page response time under 0.2 seconds
  • Content Loading: Major content loads within 2.5 seconds
  • Server Response: Code delivery within 0.8 seconds
  • Cumulative Layout Shift: Minimize unexpected webpage content shifts
  • Mobile Responsiveness: Responsive design for all device types
  • Image Optimization: Proper image sizing and aspect ratios

3.4 Technical Standards

  • HTTPS: SSL/TLS encryption for all pages
  • Domain Configuration: Both www and non-www versions function properly
  • Character Encoding: Proper UTF-8 character set implementation
  • Language Declaration: Correct HTML language attributes
  • Document Type: Proper HTML5 doctype declaration
  • Viewport Settings: Mobile-optimized viewport configuration

3.5 Security Requirements

  • Content Security Policy: Implement restrictive CSP headers
  • HSTS: HTTP Strict Transport Security implementation
  • Security.txt: Provide security contact information file
  • MIME Type Protection: Prevent MIME type sniffing
  • Clickjacking Protection: Implement frame options security
  • Error Monitoring: Code error tracking and debugging capabilities

3.6 Social Media Integration

  • Open Graph: Implement Open Graph meta tags for social sharing
  • Social Media Optimization:
    • Website title for social sharing
    • Content category designation
    • Page-specific titles and descriptions
    • Canonical URLs for sharing
    • Preview images with accessibility descriptions
